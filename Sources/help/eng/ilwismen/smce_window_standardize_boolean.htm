Standardize Boolean Input   
 

 

Standardize a boolean map:

Standardization possibilities are slightly different for factors and for constraints.

  • Standardization of factors (benefits and costs): output values range between 0 and 1;
  • Standardization of constraints; output values are either 0 or 1.

Standardization of factors (benefits and costs)

For boolean maps standardization is easy. The user only needs to specify a value to be given to the 'true' and the 'false' pixels.

Dialog box options:

Value for TRUE:

Type a value between 0 and 1 to be given to input pixels with value True. The default value to be assigned is 1.000.
Value for FALSE:

Type a value between 0 and 1 to be given to input pixels with value False. The default value to be assigned is 0.000.

 

Tip: When a factor (a benefit or a cost) with a boolean domain has been standardized, the criteria tree viewer shows information about the way the factor is standardized, see the first column below.

 

  Criterion name--Std:T=ValueT, F=ValueF i.e. True input values are assigned ValueT;
False input values are assigned ValueF

Standardization of constraints

When a map is used as a constraint, options are simplified because the user only needs to specify a condition that a pixel should satisfy in order to include it in the output.

Dialog box options:

TRUE passes, FALSE will be blocked

This means that all input pixels with value True will be included in the output map; all pixels with value False will be excluded from the output.

FALSE passes, TRUE will be blocked

This means that all input pixels with value False will be included in the output map; all pixels with value True will be excluded from the output.

 

Tip: When a constraint with a boolean domain has been standardized, the criteria tree viewer shows information about the way the factor is standardized, see the first column below.

 

  Criterion name--Std:True areas with value True in the input map will pass; areas with value False in the input map will obtain output value 0.

 

  Criterion name--Std:False areas with value False in the input map will pass; areas with value True in the input map will obtain output value 0.
