Insert a constraint

 

Insert a new constraint under the main goal of the criteria tree. Constraints are binding criteria so no compensation is allowed. Areas in an input map (added as a constraint) that do not satisfy a constraint condition, will obtain a composite index value of 0, no matter how well these areas perform in any other factor.

Factors on the other hand allow for compensation. Poor performance in one criterion can be compensated by good performance in another criterion.

The Constraint dialog box appears:

  1. when you are in the Problem Definition mode, and
  2. when you selected the main goal, or when you selected a sub goal (i.e. a group), and
  3. do one of the following:
    • choose Insert Constraint from the Edit menu in the SMCE window,
    • click the Insert Constraint button in the toolbar of the SMCE window,
    • click the main goal with the right mouse button and choose Insert Constraint from the context-sensitive menu,
    • press the Insert Key on the keyboard and choose Insert Constraint from the context-sensitive menu.

Dialog box options:

Name:

Type a new textual name for this constraint to describe it.

 

Note: Constraints will always appear under the main goal.

 

Tip: Next, select the input map or an input attribute column which will be used to calculate with this constraint.
