 Generate menu

Aggregate Values   
 

 

For each of the SMCE output maps that are currently selected, calculate:

  • the Minimum value,
  • the Maximum value,
  • the Sum of all values,
  • the weighted Average.

When you have for instance three alternative SMCE output maps, and when you selected for instance Minimum, you will obtain three Minimum values, i.e. a minimum value for each SMCE output map.

Together with the histograms of the SMCE output maps, the Aggregate Values option helps you get a general idea of the values in the SMCE output maps, and helps you to decide on the boundary values needed in Slicing.

This dialog box appears, after you selected the main goal or a sub goal in the criteria tree viewer, and:

  • clicked the Aggregate Values button in the toolbar,
  • chose the Aggregate Values command from the Generate menu, or
  • chose the Aggregate Values command from the context-sensitive menu.

 

Tip: To inspect the minimum, maximum, sum and average (mean) value in a single map, you can also use Histogram.

 

Note: Aggregate Values is not available when the SMCE output maps have not yet been generated.

Dialog box options:

Operation:

Choose to calculate for each SMCE output map within the current selection:

  • the Minimum or the Maximum value in each map, or
  • the (weighted) Average or the Sum of the values in each map.

Calculation results:

After the Calculate button has been pressed, calculation results will appear in this pane for all maps within the current selection.

Additional parameters:

Optionally, you can specify a mask map or a weight map.

  • Select this checkbox if you wish to use a mask map to reduce the study area, or to use a weight map to assign more importance to some areas, or to use a mask expression to reduce the area of interest.
  • Clear this checkbox if you simply wish to calculate Minimum, Maximum, Average or Sum without a mask or without weights.

Mask map:

Select a mask map which uses system domain Bool. The calculations will include those pixels of the SMCE output maps that have a True value in the mask map. The Mask map should use the same georeference as the SMCE output maps.

Weight map:

Select a weight map which uses system domain Value. All pixels in the SMCE output maps will be multiplied with the values in the weight map just before the calculations. Values in the weight map must be greater than zero. The weight map should use the same georeference as the SMCE output maps.

Mask Expression:

Type an expression; the calculations will include those pixels of the SMCE output maps for which the mask expression is True.

Tip:  Examples of mask expressions are for instance:
myValueMap > 0.25
(myClassMap = "pasture") or (myClassMap = "shrubs")

 

Calculate:

Click the Calculate button (or press the Spacebar on the keyboard) to start the calculations. Results will appear in the Calculation Results pane.

Additional information

Once the Minimum, Maximum, Sum or Average has been calculated, the histograms of the raster maps remain on disk.

Calculating Minimum or Maximum:

  • Undefined pixels are ignored; only pixels that are not undefined are considered.
  • The minimum value or the maximum value of all pixels in the map is shown.

Calculating Sum:

  • Undefined pixels are ignored; only pixels that are not undefined are considered.
  • The sum of all pixel values in the map is shown.

Calculating Average:

  • Undefined pixels are ignored; only pixels that are not undefined are considered.
  • The average of all pixel values in the map is shown, as:
    S (Valuei) / N
    where Valuei is the value of a pixel i, and N is the total number of pixels in the map.
  • When using a weight map, the (weighted) average of all pixel values in the map is shown, as:
    S (Valuei * Weighti) / S (Weighti)
    where Valuei is the value of a pixel, and Weighti is the weight value of that pixel.
