Create a segment map

A segment map is created in order to obtain digital line features. To obtain digital area features (polygons), you should also first digitize the lines in a segment map, and then you should use the Polygonize option in the segment editor. In the segment editor, you can use the digitizer but you can also create a segment map with the mouse.

This dialog box appears when creating a new segment map:

  • from the File menu of the Main window, choose the Create Segment Map command, or
  • by double-clicking the New SegMap item in the Operation-list, or
  • from the File menu in an existing map window, choose the Create Segment Map command. Known parameters such as coordinate system and map boundaries are already filled out. This ensures that the new segment map will fit on the map(s) displayed in the map window.

Dialog box options:

Map name:

Type a new name for the segment map.

Description:

Type a description for the map. The description of a map appears in the title bar when the map is displayed. Further, the description is visible on the status bar of the Main window when moving the mouse pointer over the map in the Catalog.

Coordinate system:

Select a coordinate system for the map; open the list box by clicking it. Or create a new coordinate system by clicking the create button: the Create Coordinate System dialog box follows.

When you work with multiple maps of the same area, you should use the same coordinate system for these maps.

Boundaries of map:

Type the minimum and maximum X and Y coordinates for the map.

  • With the minimum XY-coordinate, the XY-coordinate of the lower left corner of the map is meant;
  • with the maximum XY-coordinate, the XY-coordinate of the upper right corner of the map is meant.

The four boundary values determine the outermost coordinate values of the new map; the map cannot contain coordinates outside these values.

When you selected an existing coordinate system, the corners of the coordinate system are set as the default boundary values for this map. If the new map is just a part (e.g. one map sheet) of the total coordinate system, you should make the boundary values fit to the new map and not to the coordinate system.

Domain:

Select a domain for the map; open the list box by clicking it. Or create a new domain by clicking the create button : the Create Domain dialog box follows. A domain contains the set of valid IDs, class names or values in a map.

When creating more maps of the same theme, or when creating maps that contain the same class names, IDs or values as an existing map, select the same domain as used before. You can always add extra class names or IDs to an existing class or ID domain. Not every class name, ID or value present in a domain, needs to be used in the maps that use that domain. A representation can be linked to a domain, so that maps using the same domain can be displayed with the same colors.

The Segment editor is started: use a digitizer or the mouse to to insert segments, and assign class names, IDs, or values to the segments.

Additional information:

 

Map boundaries: It is advised to use map boundary values (Xmin, Ymin, Xmax and Ymax) according to the map rather than to a selected coordinate system.

 

Domain: The domain of a segment map defines the kind of codes that are used to name the segments:

  • (use or) create a class domain when the segments in the map belong to classes which occur in several places in the map, e.g. roads: primary roads, secondary roads, etc.,
  • (use or) create an ID domain when the segments in the map can be uniquely named, when the segments in your map are uniquely identified, e.g. individual pipe lines P8236, P9234, P9845, etc., or
  • (use or) create a value domain when the segments in the map represent measurable values, e.g. height values in a contour map.

 

It is advised to select the same domains as far as possible, for example when:

  • creating more segments maps of the same theme (Class),
  • creating more segment maps containing the same IDs,
  • creating more segment maps containing the same kind of values, or
  • creating segment maps of different years but of the same theme, or with the same class names, IDs or values.

For more information on domains, see Basic concepts : before creating maps and tables and Basic concepts : domains.
