Join Column wizard

Aggregation method

 < Back | Next >
 

This page appears in the Join Column wizard when the classes or IDs in a specified key column in the selected table (key 2) are not unique.

You have to specify the aggregation function to aggregate the values of the column that you wish to join into the current table by the groups of key 2. Optionally, you can specify a weight column to obtain weighted averages etc.

Dialog box options:

Function:

Select an aggregation function to aggregate the values of the column to be joined.

The following aggregation functions are available: Average, Count, Maximum, Median, Minimum, Predominant, Standard deviation, and Sum.

Weight:

Select this check box when you wish to use a weight column (from the selected table) during the aggregation and join. The weight column determines the relative importance of the values to be aggregated.

The weight check box is available for the following aggregation functions: Average, Median, Predominant, and Standard deviation. You can thus calculate weighted averages, weighted median values, weighted predominant values and weighted standard deviations.

When you click the Next button:

  • You will continue on the Specify output column name page of the wizard.

 

Tip: Experienced users may also join columns from other tables by typing an expression on the TabCalc command line. For more information, see Table calculation : joining columns from other tables. This topic also provides some examples, both for using the wizard and for using the command line.
