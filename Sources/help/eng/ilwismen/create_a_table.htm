Create a table

A table is for instance created to store attributes for features in existing raster, polygon, segment or point maps.

This dialog box appears:

  • when you choose Create Table from the File menu in the Main window, or
  • when you double-click the New Table item in the Operation-list, or
  • when viewing the properties of an existing map, and in the Properties dialog box, you clicked the create attribute table button.

Dialog box options:

Table name:

Type a new name for the table.

Description:

Type a description for the table. The description of a table appears in the title bar when the table is displayed. Further, the description is visible on the status bar of the Main window when moving the mouse pointer over the table in the Catalog.

Domain:

Select a domain for this table or create a new domain.
If you want to create an attribute table, you should select for the table the same domain name as is used by the map to which this table refers. Only maps of the domain type Class or ID can have an attribute table.

Only if you want to create a table that has no relation to a map, select domain None.

Records:

In case of a table with domain None, enter the number of records that the table should contain. For tables with any other domain, the domain determines the number of records that appear in the table.

The table appears in a table window. When you are in the process of creating an attribute table, all class names or Identifiers of the selected domain appear as records in the table. You can start adding attribute columns to the table.

Tip:

To create an attribute table, it is often easier to open the Properties dialog box of a map, and create a new attribute table there by clicking the create button.
