Specify a name for the output ILWIS table.

Page options:

Output Table:

Specify a name for the output table. The table will appear in the current directory.

When you used ILWIS import (i.e. when not importing a table from a database through ODBC), the output table is always in ILWIS format .

Convert to ILWIS format:

 

In case you imported a table from a database through ODBC:

  • Select this check box when you wish to obtain an ILWIS table .
  • Clear the check box when you wish to use the data from the database itself; you will obtain a Use As table u.

For more information, refer to ILWIS objects : tables - databases.

When you click the Finish button, the external file will be imported. You can inspect the result by double-clicking the output table in the Catalog.

Additional information

The following files are created:

 

.TBT:

the object definition file for the table; contains all properties. This file will be visible in a Catalog.

.TB#:

contains all the table information; i.e. the data of all fields in the table. This file is only created:

  • when you used ILWIS import to import a .DBF or an ASCII file, or
  • when you imported a table of a database, and chose to convert the table to ILWIS format.

Furthermore, when you used ILWIS import, domains and representations may be created.

Tips:

When the output table is in ILWIS table format, i.e. not for Use As tables:

  • After initial import, you can change the domain of a String column through the Column Properties of a column. Double-click a column header in the ILWIS table, and click the Create new domain from strings in column button.
  • After initial import, you can change the domain of a table through the Change domain of table operation. For instance when an imported table uses domain None, you can choose to use a class or ID column of this table to be used as the domain of the table. Then, the table can be linked to a map through the Properties of the map.
