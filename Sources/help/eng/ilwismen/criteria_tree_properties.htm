Criteria tree Properties

The Criteria tree Properties sheet lists all kind of important information on a selected criteria tree. In fact, all relevant information that is stored in the object definition file of the criteria tree is displayed. For more information, refer to Basic concepts : Properties of objects.

The Criteria tree Properties sheet appears:

  • when you click a criteria tree in a Catalog with the right mouse button, and choose Properties from the context-sensitive menu;
  • when you select a criteria tree in a Catalog, and choose Properties from the Edit menu in the Main window.

General tab:

Object type and name:

Criteria tree "Name". When you click this flat button, the criteria tree will be (re)opened.

Location:

The path and directory where this object is stored.

Object date and time:

Date and time when this criteria tree was last changed.

Object size:

Total size of the criteria tree (.SMC) on disk in bytes.

Read only:

Select this check box if this criteria tree should be read only. You are then not allowed to edit or delete the criteria tree. Clear this check box if you want to be able to edit the criteria tree.

Description:

Optionally, change the description of this criteria tree. The description is visible on the status bar of the Main window when moving the mouse pointer over the layout in a Catalog.

Tips:

  • To print or to copy the contents of all Property pages to the clipboard, choose Print or Copy from the Control menu.
  • To print or to copy the contents of the currently active Property tab to the clipboard, click with the right mouse button in the white space of the Property page and choose Print or Copy from the context-sensitive menu.
