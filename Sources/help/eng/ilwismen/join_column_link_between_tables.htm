Join Column wizard

Link between tables

 < Back | Next >
 

This page appears in the Join Column wizard when you have to select a key column from one table or from both tables to establish a link between the two tables.

Dialog box options:

Depending on the choice you made on the previous page of the wizard, the Join method page, select a column from the current table (key1), from the selected table (key2), or from both tables to create a link between the two tables.

 

Key 1:

Select, from the current table, a column that should be used to create a link between both tables. You don't have to select a key column when you use the domain of the current table. Key 1 should have a class or ID domain.

Key 2:

Select, from the selected table, a column that should be used to create a link between both tables. You don't have to select a key column when you use the domain of the selected table. Key 2 should have a class or ID domain.

 

When you click the Next button:

  • When you selected a column for Key 2, and if the classes or IDs in that column are not unique, the values values that you wish to join need to be aggregated. You will continue on the Aggregation method page of the wizard.
  • If no aggregation is needed, the Aggregation method page will not appear. You will continue on the Specify output column name page of the wizard.

 

Tip: Experienced users may also join columns from other tables by typing an expression on the TabCalc command line. For more information, see Table calculation : joining columns from other tables. This topic also provides some examples, both for using the wizard and for using the command line.
