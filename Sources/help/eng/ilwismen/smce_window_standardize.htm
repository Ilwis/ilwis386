Standardize   
 

 

Standardize the currently selected factor or constraint.

In the Problem Definition mode, you have selected input data for your criteria (factors or constraints). This input data are various maps or attribute columns with a value domain, a class domain, or the Bool domain. The values in these input maps or columns have a meaning and express different measurement units (e.g. distance, cost, age etc.). In order to compare criteria with each other in a way that makes sense, they have to be standardized, i.e. transformed to the same unit. Moreover, different standardization methods express different utility of input values. When standardizing, depending on the type of input map, a dialog box will appear in which you can choose the "rule" by which the map or column values are converted to values between 0 and 1.

Depending on the domain of the map or attribute column selected in step 2 below, one of the Standardize dialog boxes will appear:

  • For value maps: Standardize Value Input.
  • For class maps: Standardize Class Input.
  • For boolean maps: Standardize Boolean Input.

A Standardize dialog box will appear:

  1. When you are in Multicriteria Analysis mode, and
  2. when you selected a factor or constraint in the criteria tree viewer, and
  3. do one of the following:
    • choose Standardize from the Analysis menu in the SMCE window,
    • click the Standardize button in the toolbar of the SMCE window,
    • click a criteria with the right mouse button and choose Standardize from the context-sensitive menu,
    • press the Enter Key, or
    • double-click a factor or constraint in the criteria tree viewer.
