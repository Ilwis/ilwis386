Stereo Pair Properties

The Stereo Pair Properties sheet lists all kind of important information on a selected stereo pair. In fact, all relevant information that is stored in the object definition file of the stereo pair is displayed. For more information, refer to Basic concepts : Properties of objects.

This Properties sheet appears:

  • when you click a stereo pair in a Catalog with the right mouse button, and choose Properties from the context-sensitive menu;
  • when you select a stereo pair in a Catalog, and choose Properties from the Edit menu in the Main window.

General tab:

Object type and name:

Dependent Stereo Pair "Name": for a dependent stereo pair of which the output maps have been defined and/or calculated.

Stereo Pair "Name": for a stereo pair which is not dependent anymore on its input maps.

When you click this flat button, the stereo pair will be (re)opened.

Location:

The path and directory where this object is stored.

Object date and time:

Date and time when this stereo pair was last changed.

Object size:

Total size of this stereo pair (.STP) on disk in bytes. This does not include any objects which are part of the stereo pair.

Read only:

Select this check box if this stereo pair should be read-only.

Description:

Optionally, change the description of this stereo pair. The description of a stereo pair is visible on the status bar of the Main window when moving the mouse pointer over the stereo pair in a Catalog.

Dependency tab (for dependent/calculated stereo pairs):

Object type and name:

Dependent Stereo Pair "Name": for a dependent stereo pair of which the output maps have been defined and/or calculated.

When you click this flat button, the stereo pair will be (re)opened.

Definition:

In case of a dependent epipolar stereo pair:

  • A text string stating the name of the stereo pair, the names of the input raster maps from which the stereo pair has been created.
  • The expression by which an epipolar stereo pair was created (its definition) is stored internally.

In case of a dependent stereo pair from DTM, the expression by which the object was created is listed, i.e. the object definition.

Up-to-date or not:

Lists whether this stereo pair is up-to-date or not. If not, the name of the input object that was changed last is shown including that object's date and time.

Depends on:

In a tree view, the names of two raster maps are listed on which this stereo pair depends.

Calculate:

When the stereo pair and its two output raster maps have not been calculated yet:

  • click this button to (re)calculate the stereo pair and its two output raster maps. The (internal) definition of this stereo pair will be (re)executed which ensures that the object definition files of the output raster maps (.MPR) are (re)created, and that the data files of these raster maps (.MP#) are (re)calculated.

Make Up-to-Date:

When the stereo pair is not up-to-date anymore,

i.e. when one or more of the input raster maps on which this stereo pair depends have been changed, and when the output raster maps of this stereo pair are already calculated:

  • click this button to check all objects on which this stereo pair depends, see if these are up-to-date, if not, check all objects below, etc.,
  • recalculate all changed objects on which this stereo pair depends, and finally recalculate the output raster maps of this stereo pair using all new changed objects.

Release Disk Space:

When the output raster maps of the stereo pair have been calculated already:

  • click this button to delete the output raster maps of this stereo pair.

The expression, by which this dependent stereo pair was created, and by which the output raster maps can be recreated, remains stored in the stereo pair's object definition file; the output raster maps can thus always be recalculated.

Break Dependency Link:

Regardless of whether this stereo pair is Up-to-Date or not:

  • click this button to break the dependency link between this stereo pair and the input raster maps on which it depends.

Do not break the dependency link if you wish to be able to update/recalculate the output raster maps of this stereo pair in a later stage on the basis of changed input objects.

Contains tab:

Object type and name:

Dependent Stereo Pair "Name": for a dependent stereo pair of which the output maps have been defined and/or calculated.

Stereo Pair "Name": for a stereo pair which is not dependent anymore on its input maps.

When you click this flat button, the stereo pair will be (re)opened.

Objects in container:

The two output raster maps that are stored by this stereo pair are listed.

Used By tab:

Object type and name:

Dependent Stereo Pair "Name": for a dependent stereo pair of which the output maps have been defined and/or calculated.

Stereo Pair "Name": for a stereo pair which is not dependent anymore on its input maps.

When you click this flat button, the stereo pair will be (re)opened.

Used By:

In a tree view, it is listed by which other objects this stereo pair is used.

Tips:

  • To print or to copy the contents of all Property pages to the clipboard, choose Print or Copy from the Control menu.
  • To print or to copy the contents of the currently active Property tab to the clipboard, click with the right mouse button in the white space of the Property page and choose Print or Copy from the context-sensitive menu.
