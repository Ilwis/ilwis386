Create a representation

Create a new representation for an existing domain.

When you display a map with such a domain, the newly created representation is selectable in the display options dialog box of the map.

This dialog box appears when creating a new representation:

  • by opening the File menu of the Main window and choosing the Create Representation command,
  • by opening the Create item in the Operation-tree, and double-clicking the New Representation item,
  • by double-clicking the New Representation item in the Operation-list,
  • when in the Catalog, you right-click a class, group or value domain, and choose the Create Representation item from the context-sensitive menu,
  • when in a Display Options dialog box of a map with a class, group or value domain, you click the create representation button. The domain of the map, i.e. the domain for which this representation should be created, is already filled out.
  • when in the Domain Properties sheet of a class, group or value domain, on the Domain tab, you click the create representation button. The domain for which this representation should be created, is already filled out.
  • when in the Properties sheet of a map with a class, group or value domain, on the Map tab, you right-click the domain of a map and select the Create Representation command. The domain for which this representation should be created, is already filled out.

Dialog box options:

Representation name:

Type a new name for the representation.

Description:

Type a description for the representation. The description is visible on the status bar of the Main window when moving the mouse pointer over the representation in the Catalog.

Domain:

Select an existing domain for which you want to create this representation. For more information, see below.

Use percentages:

When you chose the system domain Value, select this checkbox when you want to create a representation gradual which shows values as percentages. Clear this checkbox when you want to create a representation value which shows values as values.

Representation types:

  • A representation class will be created when you select a class or group domain.
    In the Representation Class editor you can assign colors to the classes.
  • A representation value will be created when you select:
    • a user-defined value domain,
    • the system domain Value while you clear the Use Percentages checkbox, or
    • one of the other system value domains for instance system domain Image.
    In the Representation Value/Gradual editor you can assign colors to specific values by inserting and editing limits, etc.
  • A representation gradual will be created when you select the system domain Value while you select the Use Percentages checkbox.
    In the Representation Value/Gradual editor you can assign colors to percentage values by inserting and editing limits, etc.
    • A representation gradual can be used to display any value map.

Tips:

  1. The easiest way to create a representation for a map with a certain domain is to open the Display Options dialog box of that map and then click the create button next to the Representation list box.
  2. Default representations for non-system domains:
    • A new representation class will automatically become the default representation for the selected class or group domain. The representation is linked to the selected domain.
    • A new representation value will only become the default representation if you selected a user-defined value domain. The representation is linked to the selected domain.

    You can view and/or change the default representation of class and group domains and of user-defined value domains in the Properties of the domain.

  3. System domains are not editable and they therefore have a fixed default representation.
    • For system domains Value and Distance, the default representation is representation gradual Pseudo.
    • For all other system value domains, the default representation is representation gradual Gray.
