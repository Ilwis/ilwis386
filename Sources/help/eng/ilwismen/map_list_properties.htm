Map List Properties

The Map List Properties sheet lists all kind of important information on a selected map list. In fact, all relevant information that is stored in the object definition file of the map list is displayed. For more information, refer to Basic concepts : properties of objects.

The Map List Properties sheet appears:

  • when you click a map list in a Catalog with the right mouse button, and select Properties from the context-sensitive menu;
  • when you select a map list in a Catalog, and choose Properties from the Edit menu in the Main window;
  • when you open a map list, and click the Properties button in the map list viewer.

Map List tab:

Object type and name:

Map List "map list name": the map list is editable.

Dependent Map List "map list name": the map list is the result of an operation.

When you click this flat button, the map list will be (re)opened.

 

For editable map lists and for dependent map lists that have been calculated already:

Nr of raster maps:

The number of raster maps that are contained by this map list is listed.

Georeference:

The name and the description of the georeference used by the raster maps in this map list are listed.

Coordinate system:

The name, the type and the description of the coordinate system used by the georeference of the raster maps in this map list are listed.

Domain:

The name of the domain that is used by the raster maps in this map list is listed.

 

For dependent map lists (calculated or not):

Definition:

For dependent map lists: The expression by which the object was created is listed, i.e. the object definition.

 

For editable map lists and for dependent map lists that have been calculated already:

'No pyramid layers available' + Create / 'Pyramid layers available' + Remove:

 

It is listed whether pyramid layers have been created for the raster maps in this map list or not. Pyramid layers speed up the display of large raster maps in a map window.

  • If no pyramid layers are available for the raster maps, click the Create button to create the pyramid layers.
  • If pyramid layers are available for the raster maps, you can click the Remove button to delete the pyramid layers.

The Create/Remove buttons are not available when the map list and/or the raster maps are read-only.

General tab:

Object type and name:

Map List "map list name": the map list is editable.

Dependent Map List "map list name": the map list is the result of an operation.

When you click this flat button, the map list will be (re)opened.

Location:

The path and directory where this object is stored.

Object date and time:

Date and time when this map list was last changed.

Object size:

Total size of the this map list (.MPL) on disk in bytes. This does not include the raster maps that are part of the map list.

Read only:

Select this check box if this map list should be read only. You are then not allowed to edit or delete the map list. Clear this check box if you want to be able to edit the map list.

Description:

Optionally, change the description of this map list. The description of a map list appears in the title bar when the map list is displayed. Furthermore, the description is visible on the status bar of the Main window when moving the mouse pointer over the map list in a Catalog.

Dependency tab (for dependent map lists):

Object type and name:

Dependent Map List "map list name": the map list is the result of an operation. When you click this flat button, the map list will be (re)opened.

Definition:

The expression by which the object was created is listed, i.e. the object definition.

Up-to-date or not:

Lists whether this map list is up-to-date or not. If not, the name of the object which was changed last is shown including that object's date and time.

Depends on:

In a tree view, the names of the objects are listed on which this map list depends.

Calculate:

When the raster maps of this map list have not been calculated yet:

  • click this button to (re)calculate the raster maps that are contained by this map list. The definition of this map list will be (re)executed which ensures that the object definition files of the raster maps (.MPR) contained by the map list are (re)created, and that the data files of the raster maps (.MP#) in the map list are (re)calculated.

Make Up-to-Date:

When the map list is not up-to-date anymore,

i.e. when one or more objects on which this map list depends have been changed, and when the raster maps of this map lists are already calculated:

  • click this button to check all objects on which this map list depends, see if these are up-to-date, if not, check all objects below, etc.,
  • recalculate all changed objects on which this map list depends, and finally recalculate the raster maps of this map list using all new changed objects.

Release Disk Space:

When the raster maps in the map list have been calculated already:

  • click this button to delete the raster maps of this map list.

The expression, by which this dependent map list was created, and by which the raster maps can be recreated, remains stored in the map list's object definition file, so the raster maps can always be recalculated.

Break Dependency Link:

Whether this map list is Up-to-Date or not:

  • click this button to break the dependency link between this map list and the objects on which it depends. This map list will then become editable.

Do not break the dependency link if you wish to be able to update all raster map contained by the map list in a later stage on the basis of changed input objects.

Contains tab:

Object type and name:

Map List "map list name": the map list is editable.

Dependent Map List "map list name": the map list is the result of an operation.

When you click this flat button, the map list will be (re)opened.

Objects in container:

The raster maps that are part of this map list are listed.

Used By tab:

Object type and name:

Map List "map list name": the map list is editable.

Dependent Map List "map list name": the map list is the result of an operation.

When you click this flat button, the map list will be (re)opened.

Used By:

In a tree view, it is listed by which other objects this map list is used.

Additional Info tab:

Object type and name:

Map List "Name". When you click this flat button, the map list will be (re)opened.

Additional info:

When you have calculated a Variance-Covariance matrix or a Correlation matrix of this map list as input, or when you have performed a Principal Components Analysis or a Factor Analysis with this map list as input: the Optimum Index Factors (OIF) for each possible combination of three bands of the map list are listed. The Optimum Index Factors displays which combination of three bands provides the highest standard deviations with lowest correlation and can therefore probably best be viewed as a color composite. For more information, refer to How to calculate the Optimum Index Factor.

Tips:

  • To print or to copy the contents of all Property pages to the clipboard, choose Print or Copy from the Control menu.
  • To print or to copy the contents of the currently active Property tab to the clipboard, click with the right mouse button in the white space of the Property page and choose Print or Copy from the context-sensitive menu.
