Import Table wizard

 Main window, File menu

Database, Table + Table Domain

 < Back | Next >

 

Select the table from the database that you wish to use or that you wish to import, and select the domain for the table.

In case of a dBase DBF table, the table is already known, so you only need to specify a domain.

Page options:

Database tables:

Select a table from the database.

  • Excel sheets are listed with a $ sign behind the sheet name.
  • The database tables list box is not available when importing a dBase DBF file.

Possible key columns:

Select a domain for the output table.

You can select:

  • system domain None: the output table will resemble the original table.
  • a String column from the table: the items in the string column will be converted to an ID domain, and the output table will use this ID domain.

Tips:

  • For Excel import requirements, see Import Table wizard, section Import Excel .XLS requirements.
  • This option is one of the methods to access a database. For a general introduction, refer to ILWIS objects : tables - databases.
  • After initial import, you can change the domain of a String column through the Column Properties of a column.
  • After initial import, you can change the domain of a table (e.g. when table uses domain None) through the Change domain of table operation.
