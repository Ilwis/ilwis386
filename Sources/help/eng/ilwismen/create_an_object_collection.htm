Create an object collection

An object collection is a set of ILWIS objects. An object collection only stores the names and locations of the object definition files of the maps, tables, etc. that are included in the object collection. The object definition files of these maps, tables, etc. contain further references to the names and locations of the data files on disk.

Object collections:

  • can be created to organize your data as 'projects';
  • can be the result of import when only ILWIS object definition files are created while the data files remain in their native format and in their original directory.

This dialog box appears when:

  • in the Main window, you choose the Create Object Collection command from the File menu,
  • in the Operation-tree, you open the Create item, and select the New Object Collection command, or
  • in the Operation-list, you double-click the New Object Collection command.

Dialog box options:

Name:

Type a new name for the object collection.

Description:

Optionally, type a description for the object collection. The description is visible on the status bar of the Main window when moving the mouse pointer over the object collection in a Catalog.

The new object collection will be opened as a new Catalog.

You can add objects to the object collection by selecting objects in another Catalog, then drag these objects to the object collection. You can also select objects in another Catalog and then use Copy and Paste.

For more information, see How to copy objects.

Tip:

You can choose whether in a Catalog you wish to see:

  • both object collections and the objects that are included in object collections (the members of the object collections), or
  • only object collection(s) and not the members of object collections.

The setting can be specified on the Customize Catalog, Object Selection page.
