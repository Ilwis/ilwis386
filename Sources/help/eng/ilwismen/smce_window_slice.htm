SMCE window

 Generate menu

Slice ...   
 

 

Slice or classify the values in the SMCE output maps. The values in the SMCE output maps are divided over a limited number of classes. All alternatives of the SMCE output maps will be sliced. The output maps are called sliced SMCE output maps.

 

  • The number of desired output classes is specified by the user; colors are automatically assigned to the output classes, but these can be edited later.
  • By default, 5 classes of equal width are created which run from the minimum value found to the maximum value found in all alternative SMCE output maps, for instance from 0.00 to 0.20, from 0.21 to 0.40, from 0.41 to 0.60, from 0.61 to 0.80 and from 0.81 to 1.00. Values 0.20, 0.40, 0.60, 0.80 and 1.00 are called the upper boundary values of the classes.
  • However, if you wish to accentuate certain areas in the sliced SMCE output maps, you can adapt the intervals yourself (i.e. specify your own upper boundary values) by editing the Group domain that is created and used during slicing.
  • Using the same example where input values run from 0.00 to 1.00 that are divided over 5 output classes, the default output class names read 'suit. 0.00 - 0.20', 'suit. 0.21 - 0.40', 'suit. 0.41 - 0.60', 'suit. 0.61 - 0.80', 'suit. 0.81 - 1.00'; the output class names can however be edited during the slicing process.

This dialog box appears, after you selected the main goal or a sub goal in the criteria tree viewer, and:

  • clicked the Slice button in the toolbar,
  • chose the Slice ... command from the Generate menu, or
  • chose the Slice ... command from the context-sensitive menu.

 

Tip: If you wish to use your own boundary values during Slicing, it is advised to use Histogram and Aggregate Values prior to Slicing. This will help you decide on the desired boundary values for Slicing.

 

Note: Slice is not available when not all SMCE output maps have been generated.

Dialog box options:

Number of slices:

Type a value for the desired number of output classes or accept the default value. You can create any number of slices or output classes between 2 and 99.

When you accept 5 slices, the values of the SMCE output maps will be divided over 4 output classes running from the overall minimum value to the overall maximum value found in all alternative SMCE output maps.

Min, Max:

The overall minimum value found in all alternative SMCE output maps is shown, as well as the overall maximum value. If you wish, you can adapt these values.

Mind: It is not possible to exclude values smaller than the found overall minimum.

Edit output class names and boundary values:

 
  • Select this check box if you wish to edit the boundary values or the class names of the output classes. When, subsequently, the Define button is clicked, first the Domain Group editor is opened, where you can adapt the default upper boundary values (Upper Bound) and the default output class names (Class Name).
  • Clear this check box if you wish to obtain output classes of equal width and default output class names.

When you click the Show button, the sliced output maps are automatically defined, calculated and shown.

When you click the Define button, the sliced output maps will initially only be defined. After you use Show Sliced (or Show Histogram Sliced or Aggregate Values Sliced), the defined sliced maps will be automatically calculated.

The sliced output maps are automatically assigned the name of the SMCE output map from which a sliced output map is created, followed by _sliced. Besides the sliced output maps, the following objects will be created: a Group domain and a representation. These objects will obtain the name of the first SMCE output map, followed by _slices.

Tip:

For a general understanding of Group domains and Slicing, you may refer to How to create a Group domain and Slicing : functionality. The SMCE window : Slice operation integrates both the creation of the Group domain and the application of the Group domain on a number of value maps at a time.
