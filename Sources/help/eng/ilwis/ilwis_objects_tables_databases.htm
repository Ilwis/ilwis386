ILWIS objects

Tables - Databases
 

In ILWIS, you can use databases from other sources.

A database is:

  • a single file containing a single table (e.g. dBase or Paradox), or
  • a single file containing multiple tables (e.g. Excel or Access).

In general, you can access a database in two ways:

 

  • by converting parts of a database to the ILWIS table format, using the Import Table wizard, you can calculate with the output table(s), and edit the output table(s), etc. as usual.

    •  

  • by keeping the external database in its original format, and have access to tables or the entire database in a Use As manner.

    •  

    With the Use As method, you will have:

    • read access, so that the table(s) can be displayed;
    • write access, so that fields of the tables can be edited; you can add records; when you save the Use As table as a normal ILWIS table, you are also allowed to create new columns.

    When you open an entire database with the Use As method, you will obtain a so-called database collection . A database collection is similar to an object collection; the difference is that an object collection may contain all kinds of ILWIS objects (maps, tables, layouts, etc.), while a database collection may only contain tables. You are not allowed to add any tables to a database collection, and you cannot delete any tables from a database collection.

The access to an external database is achieved through a Windows technique called ADO (ActiveX Data Objects), which makes use of an OLE DB provider, which, in turn, probably uses an ODBC (Open DataBase Connectivity) driver. The OLE DB provider and the ODBC drivers that are installed on your system determine which types of databases can be used. For a standard Windows database program like Access, the required driver is already present on your system.

Using a database

Below, two methods will be shortly described to open an external database.

1. Showing and opening foreign database files through the Catalog:

This method can be used when the database consists of a single file which contains one or more tables (e.g. dBase, Paradox, Excel, Access):

 

  • Customize the Catalog so that the desired foreign extensions (e.g. dbf, xls or mdb) are shown.
  • Double-click the external file in the Catalog.

You will obtain a database collection in your Catalog. When you double-click the database collection, the tables in the database collection will appear. When you double-click a table, it will be opened in Use As manner u.

You can:

  • show tables of the database,
  • edit fields of tables in the database,
  • perform table calculations, as long as you write the output into an existing column,
  • join columns into an existing ILWIS table.

Only when you save a Use As table as a regular ILWIS table, you can also:

  • create new columns,
  • perform calculations and write the output to a new column.

Tips:

  • In a Use As table, you will find that most columns use system domain Value, or system domain String. You can convert the domain of a String column for instance by double-clicking the column's header, and then in the Column Properties dialog box, press the Create new Domain from Strings in column button.
  • To save a Use As table to a normal ILWIS table, open the Use As table and choose Save Copy As from the File menu in the table window.

2. Using the Import Table wizard:

The Import Table wizard can be used when the database consists of a single file which contains one or more tables (e.g. dBase, Paradox, Excel, Access):

 

  • From the File menu in the Main window, choose Import, Table:

    • the Import Table wizard will be opened.

  • Select the desired database.
  • Select a single table of the database, and select the domain for that table.
  • Accept the default SQL query (to retrieve all records of the table), or use your own SQL query to retrieve specific data from the table.
  • Specify an output name for the table, and choose:
    • to convert the table to the ILWIS table format, or
    • not to convert the table, but use it in Use As manner.

When you choose to convert the table to ILWIS table format, you will obtain an ILWIS table that can be shown, edited and with which you can calculate as usual.

When you choose to obtain a Use As table u, the same restrictions apply as described in section 1.

Additional information

For technical information on database collections, refer to ILWIS objects : object collections.

For your information, in Windows Control Panel, under Administrative Tools, Data Sources (ODBC), you can find a list of installed ADO/ODBC drivers.
