Map and Table Calculation

AGGMIN( ) or ColumnAggregateMin() function

Returns the minimum value of an entire column, or the minimum values per group.

Syntax

AGGMIN(ValueColumn)
AGGMIN(ValueColumn, Group)
ColumnAggregateMin(ValueColumn)
ColumnAggregateMin(ValueColumn, Group)

Input

ValueColumn is a value column
Domain type: any Value domain
Group is the 'group by' column
Domain type: any Class, ID, Group or Bool domain

Output

AGGMIN(ValueColumn) or ColumnAggregateMin(ValueColumn)

returns: 

the minimum value of a column

Domain:

the value domain of the input column

 

AGGMIN(ValueColumn, Group) or ColumnAggregateMin(ValueColumn, Group)

returns: 

a minimum value per group

Domain:

the value domain of the input column

Notes:

  • The AGGMIN and the ColumnAggregateMin function are aliases of each other. These functions can be used both in a table window and in scripts.
  • A 'group by' column can be specified when you perform an aggregation within one table. A 'group by' column must be specified when you perform an aggregation during a join.
  • The function parameters refer to nothing else than column names. This means that within the brackets no other expressions can be used.
  • Any undefined values in the input column are excluded from the calculation.

Tips:

  • On the command line of a table window, you may also use the MIN function.
  • To compare values, and find the minimum of values that are in the same record but in two or three columns, use the MIN function.
  • To compare values, and find the minimum of values that are at the same pixel position but in two or three raster maps, use the MIN function.
  • To calculate the minimum of a raster map, use the MAPMIN function.
  • When the definition symbol = is used, a dependent output column is created; when the assignment symbol := is used, the dependency link is immediately broken after the output column has been calculated.

Usable in

TabCalc

OutCol1 = AGGMIN(CommVal)

OutCol2 = AGGMIN(CommVal,Landuse)
