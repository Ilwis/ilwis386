Domain type Image

The Image domain can only contain whole values between 0 and 255. You can use the Image domain for satellite images with such values, for instance Landsat, TM and SPOT images. The Image domain is thus only available for raster maps.

All pixel values in the image represent reflection values in a certain wave-lenght interval. These pixel values are also called Digital Numbers (DN). All pixels have a value or DN; there are no undefined values.

The Image domain is available as:

  • system domain Image.

The Image domain has default representation Gray, but you can also create your own representation value, for instance to slice an image. For more information, see How to slice an image.

Tips:

  • For satellite imagery with values in another range than between 0 and 255, you can use the system domain NOAA, the system domain Radar, or any other Value domain.
  • If in images with values from 0 to 255 you want to regard value 0 as undefined, use the system domain Byte.
  • To convert maps with a certain domain to another domain, see How to change the domain of a map.
