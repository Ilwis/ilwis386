Value maps

For some operations, it is stated that a 'value map' is required or will be produced. Value maps are supposed to contain values in contrast to maps that contain for instance class names or identifiers. Values can be stored in any value domain or the Image domain. A Bool domain can only store a very limited amount of values.

When an operation needs a value map as input:

  • usually a map with a value domain or the Image domain is meant.

While performing calculations or other operations:

  • you can usually select system domain Value for the output map or output column, and
  • you can vary the value range and the precision for the values in the output map or output column during each individual operation.

You can also create your own value domains for maps and columns that contain values with a special 'standardized' meaning, for instance pH, height, yield, or concentration values. You can then create your own representation value (in values) with your default colors for this domain. When you create a user-defined value domain, it is advised to use a value range and a precision that fit your data.
