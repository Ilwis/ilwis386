Map and Table Calculation

AGGCNT( ) or ColumnAggregateCnt function

Counts the number of records which are not undefined in a column, or counts the number of records which are not undefined per group.

Syntax

AGGCNT(Column)
AGGCNT(Column, Group)
ColumnAggregateCnt(Column)
ColumnAggregateCnt(Column, Group)

Input

Column is a column
Domain type: any domain
Group is the 'group by' column
Domain type: any Class, ID, Group or Bool domain

Output

AGGCNT(Column) or ColumnAggregateCnt(Column)

returns:

a value

Domain:

system domain Count

 

AGGCNT(Column, Group) or ColumnAggregateCnt(Column, Group)

returns:

a value per group

Domain:

system domain Count

Notes:

  • The AGGCNT and the ColumnAggregateCnt function are aliases of each other. These functions can be used both in a table window and in scripts.
  • A 'group by' column can be specified when you perform an aggregation within one table. A 'group by' column must be specified when you perform an aggregation during a join.
  • The function parameters refer to nothing else than column names. This means that within the brackets no other expressions can be used.
  • Any undefined values in the input column are excluded from the calculation.

Tips:

  • On the command line of a table window, you may also use the CNT function.
  • When the definition symbol = is used, a dependent output column is created; when the assignment symbol := is used, the dependency link is immediately broken after the output column has been calculated.

Usable in

TabCalc

Examples

TabCalc example:

OutCol1 = AGGCNT(CommVal)

OutCol2 = AGGCNT(CommVal,Landuse)
