SMCE  Constraints

A Constraint in the SMCE application is a criterion that determines in the calculation of the main goal:

  • which areas should be considered as absolutely NOT suitable/performing; these areas will obtain value 0 in the final output;
  • calculation of output values will take place only in all other areas, namely according to the inserted (groups of) Factors; these areas will obtain a value between 0 and 1 in the final output.

A Constraint is either an input map or an attribute column in an attribute table.

As opposed to factors, poor performance of a constraint cannot be compensated by good performance of another factor or constraint. This leads to a nil (0) performance value in the composite index map.

When creating and filling a criteria tree:

  • constraints can only be inserted directly under the main goal in a criteria tree.
  • you need to specify for each constraint:
    • a name as text (like a description), and
    • an input map or an input attribute column (the data).

For more information, see Insert Constraint.

Before calculating the main goal (i.e. the final composite index map), you also need to:

  • standardize the input data for each constraint.

For more information, see Standardize.

 

Note: When standardizing constraints, you must select Boolean values (True or False).
