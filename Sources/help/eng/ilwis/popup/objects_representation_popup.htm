Representation   

A representation defines the manner in which the classes of a map with a class domain, or the values of a map with a value domain or the image domain should be represented on the screen and on a printer. A representation is a service object for a domain, i.e. a domain may use certain representations. Maps which use the same domain are by default displayed in the same colors.

Representation types:

  • Representation class: for raster, polygon, segment and point maps with a certain class domain, and for raster maps with a certain group domain or a certain picture domain. For each class in the domain, a representation class contains: colors for mapping units in raster maps; colors, hatching or patterns for polygons; colors, line types, line widths, etc. or equally spaced symbols for segments; colors, symbol type, symbol sizes, etc. for points.
  • Representation value: for raster, polygon, and segment maps with a certain value domain or the Image domain. A representation value contains colors assigned to specific values and to ranges of values.
  • Representation gradual: can be created for maps with the system domain Value but can be used to display any value map. A representation gradual contains colors assigned percentage values and to ranges of percentage values.
