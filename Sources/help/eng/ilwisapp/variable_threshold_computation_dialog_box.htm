Variable threshold computation

Dialog box

The Variable threshold computation operation helps you to prepare a threshold map that can be used in the Drainage network extraction operation.

You can base the threshold map on a height map (DEM), or on attribute values from an attribute table of a raster map with a class or ID domain (e.g. infiltration attribute values in a table linked to a geological unit map).

As a user, you have to:

  • specify the size of a kernel or moving window, so that for the calculation of each output pixel, only 3 x 3 or 5 x 5, etc. input pixels will be considered;
  • specify the number of desired output classes;
  • specify a boundary value and a desired output value for each output class that you wish to obtain.

When height values are used as input, you can optionally obtain an internal relief map as additional output.

For more information on the calculations that will take place, refer to Variable threshold computation : functionality.

Dialog box options:

Input DEM:

Select an input raster map. Open the list box and select the desired input map, or directly drag a raster map from a Catalog into this box.

The input map is supposed to be a Digital Elevation Model (DEM).

You can also use a class or ID map and choose a (value) attribute column from the attribute table that is linked to this map. In this way, you can base the output threshold map for instance on infiltration (attribute) values of geological units (map), etc.

Filter size:

Type an odd value to define the filter size. When you use value 3, a 3 x 3 window will move over the map(s).

If an even value is entered, the program will increase the value with 1. If a value < 3 is entered, the program will automatically use value 3.

Nr. of classes:

Select a value for the number of output 'classes' you wish to obtain. The output 'classes' are assigned the threshold values.

You can choose a value between 1 and 5.

Upper Bound:

In the first text box: specify an upper boundary value, i.e. any internal relief value smaller than or equal to this value will be assigned the threshold value.

Mind: the last upper boundary value that you specify, in the lower left box, should be large enough to handle all internal relief values, otherwise some pixels may be assigned the undefined value. Specify for the last upper boundary value instance the maximum DEM value minus the minimum DEM value, or make a liberal estimate.

Threshold:

In the text box to the right of the Upper Bound text box: specify for this upper boundary value, the output threshold value that should be assigned.

Output raster map:

Type a name for the output raster map that will contain the specified threshold values.

Description:

Optionally, type a description for the output map. The description will appear in the status bar of the Main window when moving the mouse pointer over the map or table in a Catalog, and in the title bar of a map or table window when the output map or table is displayed. If no description is supplied, the output map and the table will use their own definition as description.

Create internal relief map:

 

Select this check box when you also want to obtain an output map containing internal relief values. The output internal relief map will obtain the same name as the output threshold map, followed by __1.

Clear this check box, if you only wish to obtain an output threshold map.

When you click the Show button, the dependent output (threshold) map will be defined, calculated and shown. When you click the Define button, the dependent output threshold map will only be defined; if necessary the map will be calculated later, for instance when the map is opened to be displayed.

Optionally, a second output map containing the internal relief values can be created.
